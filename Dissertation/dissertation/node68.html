Search Spaces and States

A search space is defined as a set of candidate goal states which may or may not be a solution to an problem[#AIModernApproach##1###]. The candidate states are defined by the problem definition[#AIModernApproach##1###]. The possible states can be given by a successor function which generates new states adhering to the problem defined constraints if any[#AIModernApproach##1###].

The search space can also be explicitly defined with constraints to which a solution must adhere to[#AIModernApproach##1###]. When constraints are defined it is up to the algorithm how it moves from one state to another in the search space[#AIModernApproach##1###]. Each state the algorithm moves to is then checked whether it adheres to the constraints[#AIModernApproach##1###].

A state is a position in the search space[#AIModernApproach##1###]. This position represents a configuration that describes a possible solution to the defined problem[#AIModernApproach##1###]. When a algorithm is said to be moving to a new state, the algorithm moves to a new position in the search space[#AIModernApproach##1###]. This new position represents a different solution than the previous position that was occupied[#AIModernApproach##1###].

When dealing with an optimisation problem there can be more than one solution[#AIModernApproach##1###]. Which is why with optimisation problems not only does an algorithm have to find a solution, but also the most optimal solution among the candidate solutions[#AIModernApproach##1###]. This subset of solutions in the search space is referred to as the solution space.

Both concepts of search spaces and states have now been defined. All search algorithms operate in search spaces and occupy states. The domain of search contains a wide variety of algortihms.
