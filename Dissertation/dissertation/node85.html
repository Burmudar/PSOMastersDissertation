Cooling Schedule

The cooling schedule/annealing Schedule is the most defining characteristic of the <#2719#>SA<#2719#> algorithm. It is the procedure where the natural annealing process is mimicked. The temperature of the <#2720#>SA<#2720#> algorithm is a control parameter that defines how much the algorithm moves around in the search space.

After each iteration, whether the algorithm has selected a new best solution or not, the temperature is reduced by a certain amount. This amount is determined by the cooling schedule.

In general, when the <#2722#>SA<#2722#> algorithm temperature has a very high value most solutions that are produced from the neighbourhood are accepted [#ClusterSA##1###]. Thus the algorithm moves freely in the search space with little constraint. As the temperature decreases, the probability that the algorithm will select a bad or just any solution decreases[#ClusterSA##1###]. When the temperature is very low, the <#2725#>SA<#2725#> algorithm is similar to a greedy algorithm in the sense that it only accepts downhill movements[#ClusterSA##1###].

The cooling schedule provides the <#2727#>SA<#2727#> algorithm with the critical ability to control the rate the algorithm transitions from the diversification phase (high temperature) to the intensification phase (low temperature)[#ClusterSA##1###]. By controlling this rate, one is able to direct the algorithm to explore more early on to locate the more promosing areas for possible solutions. These promoosing areas can then be used by the algorithm in its intensification phase to find more promosing solutions.
