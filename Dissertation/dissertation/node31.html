GSM Network Management Entities

In a <#878#>gsm<#878#> network most of the elements that form part of and make the network function are often distributed in a wide geographical area to provide the best network coverage for the customer[#GSMSysEngin##1###].

For a network to function properly and efficiently network engineers need to be kept up to date on the state of the network and be alerted if any problems occur[#GSMSysEngin##1###]. For this purpose there are two systems in the <#882#>gsm<#882#> network architecture that allow for this functionality required by network engineers[#GSMSysEngin##1###].

One system is called the <#884#>OMC<#884#> which is responsible for centralised regional and local operational and maintenance activities[#GSMSysEngin##1###]. The other system is called the <#886#>NMC<#886#> and unlike the <#887#>OMC<#887#> it provides global and centralised management for operations and maintenance of the network supported by the OMCs [#GSMSysEngin##1###].
