Introduction

<#2668#>SA<#2668#> is a metaheuristic search technique proposed in the 1980s by Kirkpatrick to solve combinatorial optimisation problems. The technique is based on a natural process which is known in metallurgy as annealing [#SASingleMultiObj##1###,#TempCyclingSA##1###]. Kirkpatrick was the first to use <#2670#>SA<#2670#> to solve optimisations problems but the basic algorithm structure was defined by Metropolis et al. in 1953 [#CurveFittingSA##1###,#VeryFastSAImageEnchancement##1###].

Annealing is the natural process of crystallisation when a solid is heated to a high temperature and then systematically cooled to a lower temperature to reach a crystallised form [#NewSAs##1###,#ConstantTempSA##1###]. The crystallised form of the solid is known to be the global minimum of the solid's internal energy state.

When the solid is rapidly cooled from a high temperature, the molecules have no time to reach a thermodynamic equilibrium stage [#MobileRobotSA##1###,#ConstantTempSA##1###]. Therefore the molecules of the solid have high energy and the resultant structure has no real crystalline form; thus the solid energy is at a local minimum[#CurveFittingSA##1###,#NewSAs##1###,#MobileRobotSA##1###]. When the solid is slowly cooled in a controlled manner, the molecules are able to reach a thermal equilibrium at each temperature [#ChaosSA##1###,#CurveFittingSA##1###,#NewSAs##1###].

In the algorithm the energy state is the cost function that needs to be minimised, and the molecules are the variables, which represent the solutions, and thus their state needs to be optimised to reach the desired energy state.

The <#2678#>SA<#2678#> algorithm is able to purposely move to a worse solution in the search space[#EcoEquilSA##1###]. In the research when an algorithm moves to a worse solution it is classified as an uphill move[#AIModernApproach##1###]. Similarly, when a move is made by the algorithm which results in a better solution the move is classified as an downhill move[#AIModernApproach##1###].

The main purpose of the <#2698#>SA<#2698#> algorithm (like most optimisation algorithms) is to minimise or maximise the cost function [#SASingleMultiObj##1###]. This cost function evaluates a solution desirability compared with other solutions in the immediate neighbourhood of the algorithm's current position [#TheoPraticalSA##1###].

The immediate neighbourhood of solutions is generated based on a heuristic implemented by the algorithm designer[#AIModernApproach##1###]. This heuristic, as with the <#2703#>TS<#2703#> algorithm, can be simple or complex.

A neighbouring solution is only selected as the new best state if its desirability ranks higher than the current solution. When the algorithm moves to a better solution from the previous solution, the move is referred in the literature as a downhill move [#CurveFittingSA##1###].

The best state is not always selected; in some cases the algorithm is also able to move to solutions that are worse than the current solution. A worse solution is only selected based on a probability which is controlled by the annealing temperature of the algorithm [#TheoPraticalSA##1###].

At a high annealing temperature the probability that the algorithm will select a bad solution is very good. As the annealing temperature decreases so does the probability that a bad solution will be selected [#CurveFittingSA##1###]. When the algorithm moves to a worse solution, the move is referred to in the literature as an uphill move [#CurveFittingSA##1###]. Uphill moves allow the algorithm to break out of local minima and can lead the algorithm down a different path, which may ultimately result in obtaining the global optimum [#SASingleMultiObj##1###].

As with the <#2712#>TS<#2712#> algorithm, the standard <#2713#>SA<#2713#> algorithm does not define a set neighbourhood generation mechanism; instead it is up to the algorithm designer to implement a suitable generation mechanism that will allow the algorithm to adequately explore the search space[#VariousCoolingSA##1###].
