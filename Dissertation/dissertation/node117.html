Pheromone Update

Pheromones start to evaporate over time, and so the link marked by a pheromone trail becomes less attractive to the ants. Therefore, a path that represents a good solution needs its pheromone trail to be continuously reinforced. Certian rules govern when and by how much pheromones are reinforced.

Most of the variants that have been developed differ in what pheromone update rules they employ. In the literature pheromone update rules are classified into two groups [#CompuIntelligenceIntro##1###]. One group is called the global update rule. The other group is called the iteration-based or local update rule[#CompuIntelligenceIntro##1###].

Pheromone update rules that are in the global update group only allow the pheromone trail of the path representing the best found solution since the first iteration to be updated [#CompuIntelligenceIntro##1###]. Thus the global rule favours intensification where the algorithm exploits the global knowledge gained by the ants to find a better solution. By updating a pheromone the concentration of the particular pheromone is reinforced.

By using both types of rules the algorithm is able to efficiently exploit the history provided by the pheromones[#CompuIntelligenceIntro##1###]. The <#4006#>MMAS<#4006#> algorithm as discussed also improves on the <#4007#>AS<#4007#>. The global update rule used by <#4008#>AS<#4008#> has a disadvantage in the sense that the search might concentrate too quickly on a particular good solution (the global best path)[#FundamentalSwarm##1###]. <#4010#>MMAS<#4010#> address this disadvantage by using the global update rule on an iteration basis[#FundamentalSwarm##1###].

When considering only on an per iteration basis, the best path found by the algorithm can differ from one iteration to the next[#FundamentalSwarm##1###]. Therefore the <#4013#>MMAS<#4013#> a different path will be updating with the global update rule each iteration[#FundamentalSwarm##1###]. Using this approach the algorithm is allowed to explore the search space more[#FundamentalSwarm##1###].

Another short coming of the <#4016#>AS<#4016#> is that pheromone concentrations can become extremely high leading to less exploration by the algorithm[#FundamentalSwarm##1###]. The <#4018#>MMAS<#4018#> algorithm addresses this short coming by enforcing an maximum and minimum amount of pheromone that can exist on a path[#FundamentalSwarm##1###]. By defining a maximum the algorithm is prevents the algorithm from settling on one particular solution i.e, stagnation[#FundamentalSwarm##1###]. On the other hand, defining a minimum on all possible links between nodes ensures that links will be continiously considered for possible inclusion into a solution[#FundamentalSwarm##1###]. In addition to providing boundaries for the pheromones <#4022#>MMAS<#4022#> also uses a smoothing strategy to even out the difference between high and low pheromones[#FundamentalSwarm##1###].
